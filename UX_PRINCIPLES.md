# LOIVE UI/UX 원칙

## 레퍼런스
- **야놀자**: 카드 기반 목록, 지역/날짜 필터, 가격·평점이 한눈에 보이는 정보 밀도
- **에어비앤비**: 사진 중심 레이아웃, 호스트/리뷰 신뢰 신호, 예약 흐름의 단계별 명확성

## 4대 원칙 (모든 화면/컴포넌트 작업 시 적용)

### 1. 직관성 — 설명서 없이도 쓸 수 있어야 한다
- 아이콘만으로 의미가 통하지 않으면 라벨 텍스트를 같이 노출
- 처음 보는 사용자가 "다음에 뭘 눌러야 하는지" 헷갈리지 않게 주요 CTA는 항상 1개만 강조
- 예약/결제처럼 단계가 있는 흐름은 진행 단계를 시각적으로 표시 (예: 1/3)

### 2. 일관성 — 같은 기능은 같은 모양이어야 한다
- 버튼, 카드, 배지, 아이콘 등 컴포넌트는 재사용 (화면마다 새로 만들지 않음)
- 색상의 의미를 고정: 주요 액션=primary색, 위험/취소=error색, 완료/성공=success색 — 다른 의미로 재사용 금지
- 동일 데이터(가격, 평점, 위치)는 모든 화면에서 같은 포맷/위치 규칙으로 표시

### 3. 가독성과 접근성 — 누구나 읽고 사용할 수 있어야 한다
- 텍스트-배경 명도 대비 WCAG AA 기준(4.5:1) 이상 유지
- 터치 타겟 최소 44x44px
- 의미 전달을 색상 하나에만 의존하지 않음 (예: 에러는 빨간색 + 아이콘 + 텍스트 동시 표시)
- 폰트는 Pretendard, 본문 최소 14px 이상

### 4. 유용성과 반응성 — 빠르고 명확하게 반응해야 한다
- 모든 비동기 동작(예약, 결제, 로딩)은 로딩 상태를 즉시 표시 — 스켈레톤은 그라데이션 스윕이 아닌 opacity 펄싱(100%↔60%) 방식 사용 (DESIGN.md 스펙, 그라데이션 금지 규칙과 일치)
- 액션 결과(성공/실패)는 즉각적인 시각 피드백으로 confirm
- 네트워크 지연 시에도 버튼은 누른 즉시 반응(눌림 효과)하고, 중복 클릭은 막음 (예: 결제 버튼 더블클릭 방지)

## 적용 방식
새 화면/컴포넌트를 만들거나 리뷰할 때 이 4가지 기준으로 점검한다. 디자인 결정이 애매할 때는 야놀자/에어비앤비가 어떻게 처리했는지를 1차 기준으로 참고한다.
